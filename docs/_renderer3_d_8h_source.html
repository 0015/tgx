730 /*****************************************************************************************
731 ******************************************************************************************
732 *
733 * Drawing methods
734 *
735 ******************************************************************************************
736 *
737 * Methods to draw primitives (triangles/quads/meshes) onto the viewport.
738 *
739 * (1) The rasterizer uses the shaders set with setShaders(). If the requested shaders cannot
740 * be used because they are disabled in the template parameter DISABLED_SHADERS or because
741 * the correct parameters are not supplied (e.g. no normals for Gouraud shading or no image
742 * for texturing) then the drawing operation silently fails and return without drawing anything
743 * (in the best case, it may fall back using another shader).
744 *
745 * -> *** NORMAL VECTORS MUST ALWAYS BE NORMALIZED (UNIT LENGHT) ***
746 *
747 * -> *** TEXTURE DIMENSIONS MUST BE POWERS OF 2 WHEN USING 'FAST' WRAP MODE FOR TEXTURING ****
748 *
749 *
750 * (2) Depth testing is automatically performed when a z-buffer is available.
751 *
752 * -> *** DO NOT FORGET TO CLEAR THE ZBUFFER WITH clearZbuffer() BEFORE DRAWING A NEW SCENE ***
753 *
754 *
755 * (3) Back face culling is automatically performed if enabled (via the setCulling() method)
756 *
757 * -> *** TRIANGLES/QUADS MUST BE GIVEN TO THE DRAWING METHOD IN THEIR CORRECT WINDING ORDER ***
758 *
759 *
760 * (4) It is much more efficient to use methods that draws several primitives at once than issuing
761 * multiple drawing commands for single triangles/quads. Use the following methods by order of
762 * preferences:
763 *
764 * 1 - drawMesh() is the 'most optimized' method and the one that will usually give the faster
765 * rendering so it should be used whenever possible (for static geometry).
766 *
767 * 3 - and drawQuads() Not as fast a drawing a mesh.
768 *
769 * 3 - drawTriangles() Even slower than the methods above.
770 *
771 * 4 - drawTriangle() / drawQuad() Should only be used for drawing single triangles/quads
772 * slowest method because each draw call has an additional
773 * overhead.
774 *
775 ******************************************************************************************
776 ******************************************************************************************/

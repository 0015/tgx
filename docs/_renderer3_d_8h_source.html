550 void setViewMatrix(const fMat4& M)
551 {
552 _viewM = M;
553 // recompute
554 _r_modelViewM = _viewM * _modelM;
555 _r_inorm = _r_modelViewM.mult0(fVec3{ 0,0,1 }).invnorm();
556 _r_light = _viewM.mult0(_light);
557 _r_light = -_r_light;
558 _r_light.normalize();
559 _r_light_inorm = _r_light * _r_inorm;
560 _r_H = fVec3(0, 0, 1); // cheating: should use the normalized current vertex position (but this is faster with almost the same result)...
561 _r_H += _r_light;
562 _r_H.normalize();
563 _r_H_inorm = _r_H * _r_inorm;
564 }

884 /*****************************************************************************************
885 ******************************************************************************************
886 *
887 * Drawing methods
888 *
889 ******************************************************************************************
890 *
891 * Methods to draw primitives (triangles/quads/meshes) onto the viewport.
892 *
893 * (1) The rasterizer uses the shaders set with setShaders(). If the requested shaders cannot
894 * be used because they are disabled in the template parameter DISABLED_SHADERS or because
895 * the correct parameters are not supplied (e.g. no normals for Gouraud shading or no image
896 * for texturing) then the drawing operation silently fails and return without drawing anything
897 * (in the best case, it may fall back using another shader).
898 *
899 * -> *** NORMAL VECTORS MUST ALWAYS BE NORMALIZED (UNIT LENGHT) ***
900 *
901 * -> *** TEXTURE DIMENSIONS MUST BE POWERS OF 2 WHEN USING 'FAST' WRAP MODE FOR TEXTURING ****
902 *
903 *
904 * (2) Depth testing is automatically performed when a z-buffer is available.
905 *
906 * -> *** DO NOT FORGET TO CLEAR THE ZBUFFER WITH clearZbuffer() BEFORE DRAWING A NEW SCENE ***
907 *
908 *
909 * (3) Back face culling is automatically performed if enabled (via the setCulling() method)
910 *
911 * -> *** TRIANGLES/QUADS MUST BE GIVEN TO THE DRAWING METHOD IN THEIR CORRECT WINDING ORDER ***
912 *
913 *
914 * (4) It is much more efficient to use methods that draws several primitives at once than issuing
915 * multiple drawing commands for single triangles/quads. Use the following methods by order of
916 * preferences:
917 *
918 * 1 - drawMesh() is the 'most optimized' method and the one that will usually give the faster
919 * rendering so it should be used whenever possible (for static geometry).
920 *
921 * 3 - and drawQuads() Not as fast a drawing a mesh.
922 *
923 * 3 - drawTriangles() Even slower than the methods above.
924 *
925 * 4 - drawTriangle() / drawQuad() Should only be used for drawing single triangles/quads
926 * slowest method because each draw call has an additional
927 * overhead.
928 *
929 ******************************************************************************************
930 ******************************************************************************************/

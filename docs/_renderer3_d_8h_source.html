560 void setViewMatrix(const fMat4& M)
561 {
562 _viewM = M;
563 // recompute
564 _r_modelViewM = _viewM * _modelM;
565 _r_inorm = _r_modelViewM.mult0(fVec3{ 0,0,1 }).invnorm();
566 _r_light = _viewM.mult0(_light);
567 _r_light = -_r_light;
568 _r_light.normalize();
569 _r_light_inorm = _r_light * _r_inorm;
570 _r_H = fVec3(0, 0, 1); // cheating: should use the normalized current vertex position (but this is faster with almost the same result)...
571 _r_H += _r_light;
572 _r_H.normalize();
573 _r_H_inorm = _r_H * _r_inorm;
574 }

894 /*****************************************************************************************
895 ******************************************************************************************
896 *
897 * Drawing methods
898 *
899 ******************************************************************************************
900 *
901 * Methods to draw primitives (triangles/quads/meshes) onto the viewport.
902 *
903 * (1) The rasterizer uses the shaders set with setShaders(). If the requested shaders cannot
904 * be used because they are disabled in the template parameter DISABLED_SHADERS or because
905 * the correct parameters are not supplied (e.g. no normals for Gouraud shading or no image
906 * for texturing) then the drawing operation silently fails and return without drawing anything
907 * (in the best case, it may fall back using another shader).
908 *
909 * -> *** NORMAL VECTORS MUST ALWAYS BE NORMALIZED (UNIT LENGHT) ***
910 *
911 * -> *** TEXTURE DIMENSIONS MUST BE POWERS OF 2 WHEN USING 'FAST' WRAP MODE FOR TEXTURING ****
912 *
913 *
914 * (2) Depth testing is automatically performed when a z-buffer is available.
915 *
916 * -> *** DO NOT FORGET TO CLEAR THE ZBUFFER WITH clearZbuffer() BEFORE DRAWING A NEW SCENE ***
917 *
918 *
919 * (3) Back face culling is automatically performed if enabled (via the setCulling() method)
920 *
921 * -> *** TRIANGLES/QUADS MUST BE GIVEN TO THE DRAWING METHOD IN THEIR CORRECT WINDING ORDER ***
922 *
923 *
924 * (4) It is much more efficient to use methods that draws several primitives at once than issuing
925 * multiple drawing commands for single triangles/quads. Use the following methods by order of
926 * preferences:
927 *
928 * 1 - drawMesh() is the 'most optimized' method and the one that will usually give the faster
929 * rendering so it should be used whenever possible (for static geometry).
930 *
931 * 3 - and drawQuads() Not as fast a drawing a mesh.
932 *
933 * 3 - drawTriangles() Even slower than the methods above.
934 *
935 * 4 - drawTriangle() / drawQuad() Should only be used for drawing single triangles/quads
936 * slowest method because each draw call has an additional
937 * overhead.
938 *
939 ******************************************************************************************
940 ******************************************************************************************/

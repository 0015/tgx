526 void setViewMatrix(const fMat4& M)
527 {
528 _viewM = M;
529 // recompute
530 _r_modelViewM = _viewM * _modelM;
531 _r_inorm = _r_modelViewM.mult0(fVec3{ 0,0,1 }).invnorm();
532 _r_light = _viewM.mult0(_light);
533 _r_light = -_r_light;
534 _r_light.normalize();
535 _r_light_inorm = _r_light * _r_inorm;
536 _r_H = fVec3(0, 0, 1); // cheating: should use the normalized current vertex position (but this is faster with almost the same result)...
537 _r_H += _r_light;
538 _r_H.normalize();
539 _r_H_inorm = _r_H * _r_inorm;
540 }

860 /*****************************************************************************************
861 ******************************************************************************************
862 *
863 * Drawing methods
864 *
865 ******************************************************************************************
866 *
867 * Methods to draw primitives (triangles/quads/meshes) onto the viewport.
868 *
869 * (1) The rasterizer uses the shaders set with setShaders(). If the requested shaders cannot
870 * be used because they are disabled in the template parameter DISABLED_SHADERS or because
871 * the correct parameters are not supplied (e.g. no normals for Gouraud shading or no image
872 * for texturing) then the drawing operation silently fails and return without drawing anything
873 * (in the best case, it may fall back using another shader).
874 *
875 * -> *** NORMAL VECTORS MUST ALWAYS BE NORMALIZED (UNIT LENGHT) ***
876 *
877 * -> *** TEXTURE DIMENSIONS MUST BE POWERS OF 2 WHEN USING 'FAST' WRAP MODE FOR TEXTURING ****
878 *
879 *
880 * (2) Depth testing is automatically performed when a z-buffer is available.
881 *
882 * -> *** DO NOT FORGET TO CLEAR THE ZBUFFER WITH clearZbuffer() BEFORE DRAWING A NEW SCENE ***
883 *
884 *
885 * (3) Back face culling is automatically performed if enabled (via the setCulling() method)
886 *
887 * -> *** TRIANGLES/QUADS MUST BE GIVEN TO THE DRAWING METHOD IN THEIR CORRECT WINDING ORDER ***
888 *
889 *
890 * (4) It is much more efficient to use methods that draws several primitives at once than issuing
891 * multiple drawing commands for single triangles/quads. Use the following methods by order of
892 * preferences:
893 *
894 * 1 - drawMesh() is the 'most optimized' method and the one that will usually give the faster
895 * rendering so it should be used whenever possible (for static geometry).
896 *
897 * 3 - and drawQuads() Not as fast a drawing a mesh.
898 *
899 * 3 - drawTriangles() Even slower than the methods above.
900 *
901 * 4 - drawTriangle() / drawQuad() Should only be used for drawing single triangles/quads
902 * slowest method because each draw call has an additional
903 * overhead.
904 *
905 ******************************************************************************************
906 ******************************************************************************************/

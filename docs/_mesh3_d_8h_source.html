39 /**********
40 * Structure containing the information about a 3D mesh
41 *
42 * FORMAT:
43 *
44 * vertice the array of vertices in (x,y,z) fVec3 format.
45 * a vertex is refered by its index in this array
46 * -> maximum number of vertice: 32767
47 *
48 * texcoord the array of texture coordinate in (u,v) fVec2 format
49 * a texture coord is refered by its index in this array.
50 * Set to nullptr (and nb_texcoords=0) if no texture are
51 * used with this model.
52 * -> maximum number of texture coords: 65535
53 *
54 * normal array of normal vectors in (x,y,z) fVec3 format
55 * a normal vector is refered by its index in this array.
56 * Set to nullptr if no normal vectors are defined for this
57 * model (so only flat shading will be available).
58 * -> maximum number of normal vectors: 65535
59 * THE NORMALS MUST HAVE UNIT NORM !
60 *
61 * face The array of triangular face.
62 * The array is composed of uint16_t and is divided in
63 * "chains of triangles". See below for details
64 *
65 * texture the texture image associated with the model (if any).
66 * nullptr if texture is not present/used currently (ok
67 * even if the texcoord array exists).
68 *
69 * color the object color when texturing is not used
70 *
71 * ambiant_strength reflection factor for ambiant lightning.
72 *
73 * diffuse_strength reflection factor for diffuse lightning.
74 *
75 * specular_strength reflection factor for specular lightning.
76 *
77 * specular_exponent specular lightning component.
78 *
79 * next pointer to the next mesh to draw. nullptr if none.
80 *
81 * bounding_box model bounding box.
82 *
83 * name name of the model (optional, nullptr if none).
84 *
85 * DESCRIPTION OF THE FACE ARRAY
86 *
87 * the array is composed of "chains" of triangles. Each chain starts with a
88 * uint16_t that specifies its length and is followed by a sequence of elements
89 * where each element occupies 1, 2 or 3 uint16_t. So the array takes the following
90 * general form:
91 *
92 * [chain 1 length = n] [elem 1] [elem 2] [elem 3] ... [elem n+2]
93 * [chain 2 length = m] [elem 1] [elem 2] [elem 3] ... [elem m+2]
94 * ...
95 * [chain k length = p] [elem 1] [elem 2] [elem 3] ... [elem p+2]
96 * [endtag = 0]
97 *
98 * - An endtag (int16_t)0 is used as a sentinel to mark the end of the array.
99 *
100 * - A chain starts with a single uint16_t [chain length] that specify
101 * the number of triangles in the chain. It is followed by [chain length]+2
102 * "elements".
103 *
104 * A "element" represent a vertex of a triangle and occupies 1, 2 or 3 uint16_t
105 * depending on wether the textcoord and normal arrays exist. The first uint16_t
106 * of an element is the vertex index (with the direction bit), possibly followed
107 * by the texture index (if texture array is present) and then finally followed by
108 * the normal index (if a normal array is present) i.e an element as the form:
109 *
110 * (if texture set) (if normal set)
111 * 1bit 15bits (16bits) (16bits)
112 * [DBIT| VERTEX INDEX] [TEXTURE INDEX] [NORMAL INDEX]
113 *
114 * the highest bit of the first uint16_t is called the direction bit DBIT.
115 * Thus, a vertex index is stored on only 15 bit which is why the vertex array
116 * cannot contain more than 32767 indices.
117 *
118 * The first 3 elements of a chain represent the 3 vertices or the starting
119 * triangle of the chain. Their DBIT is always 0. Each subsequent element
120 * is combined with the current triangle to deduce the next triangle in the
121 * following way.
122 *
123 * Say the current triangle is [V1, V2, V3] and the next element is DBIT|V4:
124 *
125 * if DBIT = 0, then the next triangle is [V1, V3, V4]
126 * if DBIT = 1, then the next triangle is [V3, V2, V4]
127 *
128 * REMARK: the winding order of the triangle matters (for face culling) and
129 * that is what the DBIT is for.
130 *
131 * EXAMPLE: We assume that texcoord = nullptr but normal != nullptr so that
132 * each element is coded on 2 uint16_t: the DBIT and vertex index on the first
133 * uint16_t and the normal index on the second uint16_t.
134 *
135 * Let us consider the following face array composed of 19 uint16_t:
136 *
137 * face = {3,0,1,2,2,4,6,32773,8,7,7,1,8,7,9,4,5,5,0}
138 *
139 * It should be read as
140 *
141 * 3 (the first chain has 3 triangles)
142 * 0/1 2/2 4/6 (first triangle with vertex 0,2,4 and normals 1,2,6)
143 * 5/8 (32773 = 32768 + 5 so DBIT = 1, second triangle)
144 * 7/7 (third triangle DBIT = 0)
145 * 1 (the second chain has a single triangle)
146 * 8/7 9/4 5/5 (the triangle of the second chain)
147 * 0 (end tag)
148 *
149 * after decoding, this gives a list of 4 triangles
150 * (with normals associated with each vertex index).
151 *
152 * 0/1 2/2 4/6
153 * 4/6 2/2 5/8
154 * 4/6 5/8 7/7
155 * 8/7 9/4 5/5
156 *
157 *****/
